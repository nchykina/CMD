BWA: aligning paired reads to the reference genome

At the first step of the process the reads are aligned to the reference genome. The output of this process is a .SAM file containing information about the most likely position of every read on the genome and mapping quality.

SAMTools: converting to .bam format, sorting and indexing

Binary format is a more concise and convenient way to represent the data. Therefore, the .SAM text format is converted to .BAM binary format and indexed to make iterating over it faster.

SAMTools: performing mpileup

In the .MPILEUP file, each line represents a genomic position, consisting of chromosome name, coordinate, reference base, the number of reads covering the site, read bases, base qualities and alignment mapping qualities. Information on matches, mismatches and indels is encoded in the read base column.

BCFTools: creating a vcf file

.VCF is a text file format that contains data lines about every position in the genome where a variant is observed (SNP, Indel or CNV).

Annovar: annotating the data

Annovar annotates the variants listed in the vcf file in accordance with the publicly available databases (for instance, dbSNP). The output is generated as an .csv file that can be conveniently read by a scientist in Excel.
