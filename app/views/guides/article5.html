What is reference genome?

There are two basic kinds of tasks that are solved by bioinformaticians working with a DNA: first, one can assemble a genome de novo, and, second, one can align the reads to a reference genome. But what exactly is this reference genome? Who does it belong to, and who sequenced it? Why do we compare our reads with it, and how different is a particular living creature from this "referent" human being, mouse or dog?

Each institution has its own version of the reference genome. UCSC NCBI Ensembl? The human reference genome sequence is maintained by the Genome Reference Consortium (GRC). The current version is called GRCh37 (also referred to as hg19), older versions are NCBI36 (hg18), NCBI35 (hg17) etc. etc. Unlike the previous assemblies NCBI36 etc. GRCh37 does include some alternative regions that attempt to cover some parts of the genome known to be highly variant between individuals (the MHC region for instance). Note that the reference sequence is a mosaic haploid genome, meaning that it is assembled from mulitple individuals but only ever lists one individual's sequence at any given locus. This can have the unfortunate (but rare) effect of some regions of the reference having a haplotype which has never existed in any human population. The major browsers (Ensembl, UCSC, NCBI MapViewer) are all using the GRCh37 assembly. What is haplotype? because it is derived from just a few individuals, for part of the sequence variants the reference sequence contains the minor allele.
